## 4. TOP画面 ユーザー選択機能

### 社内ポータルへ移動ボタン
- TOP画面ヘッダー右側に**「社内ポータルへ移動」ボタン**を配置
- リンク先: 社内ポータルGAS Webアプリ（※inout-boardとは別URL。本アプリ専用のPORTAL_URLを設定）
- `target="_top"` で親フレームごと遷移
- URL管理: `Code.gs` にグローバル変数として定義し、HTMLテンプレート（`<?= PORTAL_URL ?>`）で展開
- デザイン: 緑色背景（#1f9d55）、白文字、角丸8px、ホバー時に濃い緑（#177a43）

### メイン検索窓（主導線）
- **お弁当箱型デザインの検索窓**を画面中央に大きく配置
- お弁当箱の蓋が開いているようなデザインで、中に名前を入力する
- 名前を直接入力して検索できる（インクリメンタルサーチ）
- 入力候補をドロップダウンリストで表示し、候補クリックで直接カレンダーを開く
- ※ マスタデータは初回ロード時に全件取得し、クライアント側でフィルタ

### プルダウン選択（補助導線・検索窓の上部に配置）
- 「部署から探す場合はこちら」のラベルを添える
- **部署プルダウン:** 左側に配置。部署（機械設計 / TSC / 製缶・溶接 / 機械加工 / 組立・塗装）から選択
- **名前プルダウン:** 部署プルダウンの右横に配置。選択した部署に紐づく社員のみ表示される（連動フィルタ）

### ユーザー選択フロー
1. 方法A（推奨）: 検索窓に名前を直接入力 → 候補ドロップダウンから選択
2. 方法B（補助）: 部署プルダウンで部門を選択 → 名前プルダウンに該当部署の社員のみ表示 → 名前を選択

※ いずれの方法でも自分の名前を選んでもらい、個人を特定してから予約画面へ進む

### 認証・セキュリティ方針
- ログイン認証は行わず、**性善説で運用**する（名前選択方式）
- ただし`Session.getActiveUser()`でGoogleアカウント情報を取得し、操作ログに記録する
- 操作ログ: 誰が・いつ・どの予約を変更したかをバックエンドシートに記録

## 5. 予約カレンダー画面

### カレンダー表示（お弁当箱グリッド型）
- ユーザー選択後、**大きなポップアップ（モーダル）でカレンダーを表示**
- 画面中央にオーバーレイで大きく表示し、視認性・操作性を重視
- 月の16日～翌月15日の期間を表示（手配期間に対応）
- 月のまたぎ部分（例: 4月分と5月分）に**月ラベル（薄い背景色の区切り行）**を追加
- **今日の日付セルに濃いアンバー色の太枠ハイライト**を付ける
- 社内カレンダーマスタから休日情報を取得し、休日はカレンダー上でグレーアウト（注文不可）にする
- 背景は薄暗くし、カレンダーに集中できるようにする
- ポップアップ外クリックまたは閉じるボタンで閉じる
- モーダル閉じる際、変更があった場合は**「保存しました」のトースト通知**をTOP画面上に表示

#### カレンダーデザイン
- カレンダー全体が**大きなお弁当箱**の形をしている
- 各日付セルが**おかずの仕切り（小さな四角い区切り）**のようなデザイン
- ○（弁当注文）の日: 仕切りの中に**おにぎりや卵焼きの小さなアイコン**が入る
- 「お」（おかずのみ）の日: 仕切りの中に**おかずアイコン**が入る
- ー（注文なし）の日: **空の仕切り**
- 休日: 和柄の薄い模様入りグレーの仕切り
- 色合い: **木目調のお弁当箱ベース**、仕切りは**淡いクリーム色**
- TOP画面の弁当箱検索窓と統一感のあるデザイン

### 予約可能期間の制限
- 予約（○/お）の入力・変更ができるのは**本日から2週間以内の日付のみ**
- 2週間より先の日付はグレーアウトし、クリックしても反応しない
- 過去の日付も変更不可（確定済み）
- **当日分は AM9:00 を過ぎると変更不可**
- **出勤土曜分は前日（金曜）15:00 を過ぎると変更不可**
- 締切後の注文は「総務へ直接ご連絡ください」と案内表示
- ※ 全日程を一括で○にするような不正利用を防止する目的

#### 締切判定の方式
- **締切判定はクライアント側JavaScript（端末時刻）で行う**
- サーバー側（9時トリガー）は集計・メール送信のみ担当
- ※ GASの時間ベーストリガーは9:00~9:59の間で実行されるため、正確な9:00制御はクライアント側が担う

### カレンダー上部の集計表示（個人サマリー）
- カレンダーポップアップの上部に、当月度（16日～翌月15日）の個人集計を表示
- 表示内容:
  - **{月}度　弁当: ○個 / おかずのみ: ○個 / 合計: ○個**
- 内訳:
  - **予定数:** 未来日の予約分も含めた合計
  - **実績数:** 本日以前で確定済みの実績数
- 例: 「5月度　弁当: 12個（実績8 + 予定4） / おかずのみ: 3個（実績2 + 予定1） / 合計: 15個」

### UI保存状態表示
- **モーダル内（蓋ヘッダー付近）**に保存状態インジケーターを配置
- **保存済み** / **保存中** / **未保存** / **エラー** / **オフライン** の5状態を表示

### 日付クリックによる注文切替（3状態トグル）

| クリック回数 | 表示 | 意味 |
|---|---|---|
| デフォルト | **ー** | 注文なし |

| 3回クリック | **ー** に戻る | 注文キャンセル（デフォルトに戻る） |

### 配達拠点の指定機能

#### 背景
- 組立員・事務職員の中に、拠点を兼務する者がいる
- 例: 本社工場所属だが、ある日は新工場で勤務 → 新工場に届けてほしい

#### デフォルト拠点
- 作業員マスタの「拠点」をデフォルト値として各日に自動設定
- 通常はデフォルトのまま変更不要

#### 拠点変更の操作フロー
1. カレンダー上に**「拠点変更」ボタン**を設置（通常は非活性/目立たない状態だが視認できるレベル）
2. 「拠点変更」ボタンを押すと、拠点変更モードに切り替わる
3. 拠点変更モード中、各日付セルをクリックすると**拠点の切替**が可能（新工場 ⇔ 本社工場）
4. 変更後の拠点は視覚的に区別できるようにする（色分け・アイコン等）
5. 再度「拠点変更」ボタンを押すと通常モード（注文切替）に戻る

#### 誤操作防止の配慮
- 通常の注文操作（○/お/ー）と拠点変更は**モードを明確に分離**
- 拠点変更ボタンを押さない限り、拠点は変更できない
- 拠点変更モード中はモーダル全体の枠色を青系に変え、ヘッダーに**「拠点変更モード中」のバナー**を表示

#### 拠点バッジ
- 各日付セル右下に拠点バッジを表示（最低10px以上、📍新 / 📍本）
- 工場スタッフ: 新工場 ⇔ 本社工場
- 事務職員: 新工場事務所 ⇔ 本社事務所

### データ保存方式（A+C+B方式 ※inout-board準拠）

inout-boardアプリ（https://github.com/skyblueearthjapan/-inout-board.git）と同じ3層保存方式を採用する。

| 層 | 保存先 | タイミング | 目的 |
|---|---|---|---|
| **A層 (Draft)** | localStorage | クリック即座 | クラッシュ・ブラウザ閉じ時のリカバリ用 |
| **C層 (ChangeQueue)** | メモリ内キュー | 変更を集約 | 複数日の変更をまとめる |
| **B層 (Backend)** | スプレッドシート | 3秒デバウンス + 30秒定期保存 | 永続保存 |

#### A層フォールバック
- localStorage が利用不可（iFrame制約・ブラウザ設定等）の場合は sessionStorage にフォールバック
- それも不可の場合はA層を無効化し、C層→B層のみで動作（クラッシュリカバリなし）

#### 保存データ構造（1レコードあたり）
- 作業員コード（ユーザーID）
- 氏名
- 日付
- 注文状態（ー / ○ / お）
- **配達拠点（新工場 / 本社工場）** ※デフォルトはマスタの拠点、日別変更があれば上書き
- **最終更新日時（タイムスタンプ）**

#### 通信方式
- `google.script.run` でGASサーバー関数を呼び出し
- google.script.runをPromise化するラッパーを実装
- 一括更新API（`api_applyPatch`相当）で複数変更をまとめて保存
- シーケンス番号で送信順序を管理し、二重書き込みを防止

#### 排他制御
- `LockService.getDocumentLock()` でスプレッドシートへの同時書き込みを防止
- ロック取得失敗時: 最大3回リトライ（2秒間隔）、それでも失敗時はユーザーにエラー表示

#### 楽観的UI
- クリック時、即座に画面上の表示が切り替わる（サーバー応答を待たない）
- バックグラウンドでB層へ保存処理を実行
- 保存失敗時のみエラー通知を表示し、表示を元に戻す（C層にundo情報を保持）

#### オフライン・リカバリ対応
- ネットワーク復帰時にA層のDraftデータを自動でサーバーへ再送信
- ページ離脱時、未保存変更がある場合は警告を表示
- ※ GAS WebアプリはService Worker不可のため、新規アクセスでのオフライン表示は非対応

## 7. UI/デザイン方針

### コンセプト
- 小さくいろんな食べ物のイラスト/アイコンをちりばめた**かわいらしいデザイン**
- ただしPOPすぎず、派手すぎない**シックさ**を兼ね備える
- 下品にならず**上品さ**を重視する

### デザインガイドライン
- **トーン:** かわいい × 上品（カジュアルすぎない、フォーマルすぎない）
- **食べ物モチーフ:** おにぎり、お弁当箱、お箸、卵焼き、おかず等の小さなイラストを装飾的に配置
- **配色:** 落ち着いた暖色系ベース（和食を連想させる上品な色合い）
- **フォント:** 丸みがあるが読みやすい、親しみのあるフォント（Zen Maru Gothic等）
- **余白:** 十分な余白を確保し、ゴチャつかせない

### レスポンシブ対応
- PC・タブレット・スマートフォンに対応
- カレンダーセルは最低44px以上のタップ領域を保証
- プルダウン・集計エリアは狭い画面で自動的に縦積みレイアウトに切り替え

| 備考 | 祝日名等 | 昭和の日, 憲法記念日 等 |

### カレンダー上の表示ルール

| 区分 | カレンダー表示 | 注文可否 |
|---|---|---|
| 平日（マスタに記載なし） | 通常表示 | 注文可能 |
| **出勤土曜** | 通常表示（土曜色） | **注文可能** |
| **休日** | グレーアウト | 注文不可 |
| **祝日** | グレーアウト＋祝日名表示 | 注文不可 |
| マスタに記載のない土曜日 | **休日扱い（グレーアウト）** | 注文不可 |
| 日曜日 | **休日扱い（グレーアウト）** | 注文不可 |

- GASトリガーで毎日自動取得（マスタ同期トリガーは朝8時台に実行し、9時の集計前に完了させる）
